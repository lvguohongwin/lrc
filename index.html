介绍

项目中包括了 3 个文件

  • .css 要自定义 配合 js 就可以了
  • main.js
  • index.html 演示文件

灵动的力量

将js 文件导入播放器页面，注意默认的是HTML5 中的audio 或 vedio 标签，如果是其他，还需要使用之配置一个适配器了 一般的将文件最后导入 为妙，
引入之后，只需要 4 行代码就可搞定

  • lcz.set;rc_objec('歌词显示标签的id')
  • lcz.mf_Init();
  • lcz.load(lcz.Read('歌词原格式'，解析器)，回调函数，也就是解析完必之后的第一个处理函数)
  • 与播放器挂钩： 在播放器的 ontimeupdate 事件中加入一个函数 lcz.torun();

其次：

工具中自动 带了 两个解析器：

1. Read(); 简单型： 解析单时间格式歌词 如： [00:12.30]歌词
2. vls1(); 加强型： 解析多时间格式歌词 如： [00:00.45][10.50.20].. 歌词

同样的有两个 时间追踪器

1. getLrcByVlue(var); // 相对于两者，反应较慢
2. getLrcByValueInd(var); //相对于两者，反应快 （在歌曲快进，快退是表现明显）

详细的变量设置

          classV1="v1"; //歌词原始样式  (该样式名称 使用者可以自定义，只需指定名称就行) 
       classV2="v2";    //正在播放歌词的样式
     	initTop = 200;    //初始化显示位置 200 px    值越大上边距越小
     	center = 150;     //歌词焦点中心位置   50 px 
     	empty = 20;       //初始化构建多少个空对象 （ 播放前的歌词显示位置）    同样也可以调整上边距，值越小上边距越小
     	isDropLrc = true;  //是否启用本地歌词 拖入功能 （浏览器必须支持拖动）
     		// 解读歌词过程中用到的 标记
     	tag = {
         	sname :"ti",    //ti表示   歌曲名     
         	cname :"cl",    //ci表示   作词
         	qname :"cs",    //cs表示   作曲
         	bname :"ps",    //ps表示   编曲
         	sgname :"ar",   //ar表示   歌手名称
         	special :"al",  //al表示    所属专辑
         	kname :"by",		/by表示    歌词制作
         	end :"end"      //end表示   结束 语
     	};
